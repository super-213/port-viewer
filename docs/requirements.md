# Port Viewer 产品需求文档（PRD）

> 文档版本：v0.1（初稿）  
> 文档状态：已确认  
> 更新日期：2026-07-17  
> 产品形态：macOS 原生应用 + 菜单栏工具  
> 建议技术方向：Swift / SwiftUI，底层基于系统自带的 `/usr/sbin/lsof`

## 1. 产品概述

Port Viewer 是一款面向 macOS 用户的原生端口检查工具。应用将 `lsof` 的端口查询能力包装为易理解的图形界面，让不熟悉命令行的用户也能快速回答以下问题：

- 某个端口是否被占用；
- 哪个应用或进程占用了端口；
- 当前有哪些 TCP/UDP 端口与连接；
- 是否可以安全结束占用进程并释放端口；
- 不打开主窗口时，能否从菜单栏快速查看和处理端口占用。

产品应保持“系统工具”而非“复杂运维平台”的定位：启动快、信息清楚、操作可信、默认安全，并在视觉与交互上遵循 macOS 原生习惯。

## 2. 背景与问题

开发者、本地服务使用者及普通用户经常遇到“端口已被占用”的错误。命令行通常需要组合使用 `lsof`、`ps` 和 `kill`，用户还要理解 PID、TCP 状态和信号等概念，门槛较高。已有命令行方案还存在以下问题：

- 命令和参数不易记忆；
- 输出字段密集，普通用户难以识别真正的占用者；
- 结束错误进程可能造成数据丢失或系统异常；
- 端口状态会快速变化，静态输出容易过期；
- 高频检查时，反复打开终端效率较低。

Port Viewer 通过结构化展示、搜索筛选、状态解释、操作确认和菜单栏快捷入口降低上述成本。

## 3. 产品目标

### 3.1 核心目标

1. 用户能在 5 秒内查明指定端口的占用进程。
2. 用户无需编写命令即可查看本机 TCP/UDP 端口与网络连接。
3. 用户能在明确风险提示下结束目标进程，并确认端口是否已释放。
4. 用户可以从菜单栏快速查看端口概况、搜索端口和打开详情。
5. 界面应简洁、克制、有质感，并与 macOS 系统界面保持一致。

### 3.2 非目标（首版不做）

- 不提供远程主机扫描、局域网扫描或公网端口探测；
- 不替代防火墙、抓包工具或完整网络诊断套件；
- 不提供进程启动、服务编排、端口转发或代理配置；
- 不提供 Windows/Linux 版本；
- 不采集或上传用户的进程、端口与连接数据；
- 不执行用户输入的任意 Shell 命令；
- 不承诺结束所有系统进程或受系统保护的进程。

## 4. 目标用户与典型场景

### 4.1 目标用户

- 初级开发者：遇到 `Address already in use`，但不了解 `lsof` 和 `kill`；
- 专业开发者：需要频繁查看本地服务、模拟器、容器或调试进程占用的端口；
- 测试与技术支持人员：需要快速确认本机服务是否监听指定端口；
- 普通用户：希望判断某个应用是否正在建立网络连接。

### 4.2 典型场景

1. 用户输入 `3000`，发现 Node 进程正在监听，结束进程后重新启动项目。
2. 用户从菜单栏查看当前监听端口数量，并快速搜索 `8080`。
3. 用户按进程名搜索 `python`，查看该进程使用的全部端口。
4. 用户看到非本人进程占用端口，应用说明权限不足并提供可理解的处理路径。
5. 用户结束进程失败，应用显示失败原因，而不是误报“端口已释放”。

## 5. 产品原则

1. **先解释，再操作**：向用户展示“进程、端口、协议、影响”后再允许结束进程。
2. **危险操作可预期**：结束进程必须确认；强制结束需要二次升级确认。
3. **状态必须可验证**：操作后重新查询端口，依据实际结果反馈成功或失败。
4. **原生优先**：使用系统控件、系统字体、系统语义色、标准快捷键和辅助功能。
5. **信息密集但不拥挤**：主窗口适合比较和筛选，菜单栏只放高频信息和动作。
6. **隐私优先**：查询在本地执行，默认无账号、无云端同步、无网络上传。

## 6. 范围与优先级

优先级定义：P0 为首个可用版本必须具备；P1 为首版后优先补充；P2 为远期候选。

| 模块 | 能力 | 优先级 |
|---|---|---:|
| 端口查询 | 展示本机 TCP/UDP 端口与连接 | P0 |
| 搜索 | 按端口、PID、进程名搜索 | P0 |
| 筛选排序 | 协议、TCP 状态、用户、监听状态筛选与列排序 | P0 |
| 进程详情 | 展示进程及其端口/连接明细 | P0 |
| 释放端口 | 温和结束进程、失败后可选择强制结束 | P0 |
| 菜单栏 | 状态项、概况、搜索、快捷打开与退出 | P0 |
| 自动刷新 | 可配置刷新间隔，支持暂停 | P0 |
| 系统适配 | 浅色/深色、VoiceOver、键盘操作、减少动态效果 | P0 |
| 偏好设置 | 刷新频率、菜单栏指标、启动行为 | P1 |
| 登录启动 | 登录时自动启动 | P1 |
| 关注端口 | 收藏常用端口并突出状态变化 | P1 |
| 通知 | 关注端口被占用或释放时发送本地通知 | P1 |
| 历史记录 | 仅在本地保存端口状态变化 | P2 |
| 导出 | 导出当前列表为 CSV/JSON | P2 |

## 7. 信息架构

### 7.1 主窗口

主窗口建议采用 macOS 原生工具型布局：

```text
┌────────────────────────────────────────────────────────────────────┐
│ [侧栏]  Port Viewer        [搜索端口、PID 或进程]  [暂停] [刷新] [设置] │
├──────────────┬─────────────────────────────────────────────────────┤
│ 全部         │ 监听 24 · 活跃连接 61 · 最后更新 10:42:18            │
│ 监听端口     ├─────────────────────────────────────────────────────┤
│ 活跃连接     │ 进程       PID   协议   本地端点     状态      用户    │
│ UDP          │ node       4120  TCP    *:3000      LISTEN    me     │
│ 关注的端口   │ postgres   928   TCP    127.0.0.1:5432 LISTEN me     │
│              │ ...                                                 │
│              ├─────────────────────────────────────────────────────┤
│              │ 选中项详情：进程、路径、端点、风险说明与操作          │
└──────────────┴─────────────────────────────────────────────────────┘
```

窗口区域：

- **工具栏**：全局搜索、自动刷新暂停/继续、立即刷新、设置入口；
- **侧栏**：全部、监听端口、活跃连接、UDP、关注的端口（P1）；
- **概况行**：监听数、活跃连接数、更新时间和查询状态；
- **主列表**：可排序、可调整列宽的原生表格；
- **详情区**：选中记录的完整信息、风险提示及“结束进程”操作。

### 7.2 菜单栏弹出面板

点击菜单栏状态项后展示轻量面板，建议宽 360–400 pt、高度不超过 520 pt：

```text
┌────────────────────────────────┐
│ Port Viewer      已更新 10:42  ↻ │
│ [ 搜索端口或进程…              ] │
│ 监听 24             活跃连接 61 │
├────────────────────────────────┤
│ :3000  node              LISTEN │
│ :5432  postgres          LISTEN │
│ :8080  java              LISTEN │
│ ...                            │
├────────────────────────────────┤
│ 打开主窗口     设置       退出   │
└────────────────────────────────┘
```

菜单栏面板要求：

- 默认展示监听端口概况和最近排序的若干条记录；
- 支持按端口号或进程名即时过滤；
- 点击记录打开主窗口并定位到对应项；
- 支持手动刷新；
- 查询失败或已暂停时在状态项与面板内明确显示；
- “结束进程”若放入菜单栏面板，也必须使用确认对话框，不能单击直接执行；
- 关闭主窗口后应用继续在菜单栏运行；用户选择“退出 Port Viewer”才完全退出。

## 8. 功能需求

### 8.1 查询与刷新

#### FR-001 初次查询

- 应用启动后立即执行一次本机端口查询；
- 查询期间显示不阻塞交互的加载状态；
- 主线程不得直接运行 `lsof` 或解析大量输出；
- 查询完成后一次性更新快照，避免列表逐行跳动；
- 首次查询无结果时应显示空状态，而不是错误状态。

#### FR-002 自动刷新

- 主窗口可见时按固定间隔自动刷新；建议默认 3 秒；
- 菜单栏常驻但主窗口关闭时可使用较低频率；建议默认 5 秒；
- 用户可暂停/继续自动刷新；
- 用户可随时触发立即刷新；
- 同一时间只允许一个查询任务运行，新刷新请求不得无限堆积；
- 刷新时保留当前搜索、筛选、排序和尽可能保留选中项；
- 设置页可在 P1 中提供刷新频率选项，例如 1、3、5、10 秒和仅手动。

#### FR-003 查询状态

应用至少区分以下状态：

- 首次加载；
- 数据正常且自动刷新中；
- 自动刷新已暂停；
- 查询失败但仍保留上一份成功快照；
- 尚无数据；
- `lsof` 不可用或输出无法解析。

失败时应显示最近一次成功更新时间、简明原因和“重试”动作，不应直接清空旧数据。

### 8.2 数据展示

#### FR-004 列表字段

P0 列表至少包含：

| 字段 | 说明 |
|---|---|
| 进程 | `lsof` 返回的 command；可获取时显示应用图标 |
| PID | 进程标识 |
| 用户 | 进程所属用户 |
| 协议 | TCP / UDP，并区分 IPv4 / IPv6（可放入详情） |
| 本地地址 | 如 `127.0.0.1`、`*`、`[::1]` |
| 本地端口 | 保持数字显示，不转换为服务名称 |
| 远端地址/端口 | 无远端时显示 `—` |
| 状态 | 如 LISTEN、ESTABLISHED；UDP 无状态时显示 `—` |
| 文件描述符 | 放入详情，便于专业用户排查 |

默认视图建议聚焦“监听端口”。“全部”视图可展示监听端口与已建立/其他网络连接。

#### FR-005 分组与重复

- TCP 与 UDP 的同号端口是两条不同记录；
- IPv4 与 IPv6 记录应准确保留，可在视觉上合并展示，但详情中必须可区分；
- 同一进程的多个文件描述符或连接不得因简单去重而丢失；
- 同一端口由多个进程共享时，应展示全部占用者；
- 列表稳定标识建议由 PID、文件描述符、协议、本地端点、远端端点组合生成。

#### FR-006 详情

选中记录后，详情至少展示：

- 进程名、PID、用户；
- 本地端点、远端端点、协议、状态；
- 文件描述符；
- 可获取时展示可执行文件路径、父进程和应用图标；
- 同一进程的其他端口与连接；
- 当前记录的更新时间；
- “结束进程”按钮与风险说明。

### 8.3 搜索、筛选与排序

#### FR-007 搜索

- 单一搜索框同时支持端口号、PID 和进程名；
- 纯数字默认同时匹配完整端口号与 PID，完整匹配排在前面；
- 文本搜索对进程名不区分大小写；
- 输入 `:3000` 时只匹配端口 3000，避免与 PID 混淆；
- 输入时即时过滤已有快照，不应为每个字符重新执行 `lsof`；
- 无匹配结果时说明当前筛选条件，并提供清除操作；
- `⌘F` 聚焦搜索框，`Esc` 清空搜索或退出搜索状态。

#### FR-008 筛选

P0 筛选项：

- 范围：全部、仅监听、活跃连接；
- 协议：全部、TCP、UDP；
- IP 版本：全部、IPv4、IPv6；
- 用户：全部、当前用户、其他用户；
- TCP 状态：LISTEN、ESTABLISHED 及查询结果中出现的其他状态。

筛选条件可组合；存在筛选时应有清晰标识并支持一键重置。

#### FR-009 排序

- 支持按进程名、PID、端口、协议、状态和用户排序；
- 默认先按本地端口升序，再按进程名排序；
- 刷新后保持用户选择的排序方式；
- 端口按数值而不是字符串排序。

### 8.4 结束进程与释放端口

#### FR-010 操作入口

- “结束进程”只在选中有效记录后启用；
- 按钮使用明确文字，不使用只有专业用户才理解的 `kill` 作为主标签；
- 键盘快捷键不得在无确认情况下直接结束进程；
- 不提供“直接关闭端口”的误导性表述，因为实际操作是向占用进程发送信号。

#### FR-011 标准结束流程

1. 用户点击“结束进程”；
2. 应用立即重新查询/校验目标，确认 PID 仍存在且仍占用所选端口，降低陈旧快照和 PID 复用风险；
3. 显示确认对话框，至少包含进程名、PID、端口、协议，以及“该进程的其他端口也会一并关闭”的提示；
4. 用户确认后优先发送 `SIGTERM`，允许进程清理资源并正常退出；
5. 短暂等待后重新执行端口查询；建议等待 1–2 秒；
6. 若目标端口已释放，显示成功反馈；
7. 若进程仍存在或端口仍被占用，说明现状并允许用户选择“强制结束”；
8. “强制结束”发送 `SIGKILL` 前必须再次确认，并说明可能导致未保存数据丢失。

#### FR-012 多进程占用

- 若同一端口存在多个占用进程，确认框应列出全部进程；
- 默认仅结束用户明确选中的进程；
- “结束该端口的全部占用进程”不纳入 P0，除非后续确认确有必要；
- 操作后根据实际查询区分“进程已结束但端口仍被其他进程占用”与“端口已完全释放”。

#### FR-013 权限与受保护进程

- 当前用户拥有的普通进程可直接请求结束；
- 非当前用户、root 或系统受保护进程可能需要管理员权限，也可能无法结束；
- 权限不足时不得反复弹出认证框或笼统显示“未知错误”；
- 提示应说明需要何种权限、目标是什么，并允许取消；
- 对关键系统进程应增加醒目但克制的风险提示；
- 管理员授权实现方式需在技术设计阶段单独评审，授权范围必须限制到目标 PID 和指定信号。

#### FR-014 结果反馈

结果至少区分：

- 端口已释放；
- 原进程已结束，但端口被另一个进程立即重新占用；
- 进程拒绝或无法接收信号；
- 权限不足；
- 进程已在操作前自行退出；
- 操作超时；
- 查询失败，暂时无法验证端口状态。

成功或失败均应提供简短、可读的反馈；错误信息需附带下一步动作，如重试、打开详情或使用管理员权限。

### 8.5 菜单栏

#### FR-015 状态项

- 应用运行时在菜单栏显示单色模板图标，自动适配浅色/深色菜单栏；
- 可选在图标旁显示一个指标，建议默认为监听端口数量；
- 查询失败、暂停或正在刷新时不能只靠颜色表达，可通过图标形态或文本辅助表达；
- 指标过大时应避免占用过多菜单栏空间。

#### FR-016 快捷面板

- 点击状态项打开弹出面板；
- 显示监听数、活跃连接数、更新时间；
- 提供搜索、立即刷新、打开主窗口、设置与退出；
- 列表默认限制为 8–12 条，更多内容跳转主窗口；
- 面板中的当前搜索与主窗口搜索是否同步，列入待确认项；
- 面板应支持键盘导航和 VoiceOver；
- 点击面板外关闭时不影响后台刷新。

### 8.6 设置

P1 设置项建议包括：

- 主窗口与菜单栏后台刷新频率；
- 菜单栏显示：仅图标 / 监听端口数 / 关注端口状态；
- 启动时是否打开主窗口；
- 关闭最后一个窗口后是否继续在菜单栏运行；
- 登录时启动；
- 外观：跟随系统 / 浅色 / 深色（如确有必要；首选仅跟随系统）；
- 是否显示系统进程；
- 危险操作确认策略不可完全关闭，最多可降低普通 `SIGTERM` 的提示强度。

## 9. 底层查询与数据要求

### 9.1 `lsof` 调用原则

- 使用系统绝对路径 `/usr/sbin/lsof`；
- 使用 `Process`/参数数组调用，不通过 Shell 拼接命令；
- 使用 `-nP` 禁止主机名解析和端口服务名转换，保证速度与端口数字准确性；
- 查询 TCP 与 UDP 网络记录；
- 优先使用 `lsof` 的机器可读字段输出（`-F`，建议 NUL 分隔）而不是按空格拆分表格；
- 解析器必须容忍未知字段、字段缺失、IPv6 地址、进程在查询中途退出等情况；
- 具体字段组合和 macOS 各目标版本输出差异需由技术方案与自动化样本测试确定；
- 应记录可诊断但不含敏感连接内容的本地错误日志，默认不上传。

建议基线查询思路（仅作技术验证起点，不作为最终固定命令）：

```text
/usr/sbin/lsof -nP -iTCP -iUDP -F0...
```

### 9.2 数据解释

- “监听端口”主要对应 TCP `LISTEN`；
- UDP 通常没有与 TCP 相同的 LISTEN 状态，应单独归类，不能错误标记为 LISTEN；
- `*`、`0.0.0.0`、`::` 表示的绑定范围需要用易懂文案解释；
- 端口记录是某一时刻的快照，不应暗示其永久有效；
- UI 中的“释放端口成功”必须以操作后的查询结果为准。

### 9.3 降级与兼容

- 找不到 `lsof`：显示应用无法查询的原因和修复建议；
- `lsof` 返回非零退出码但有部分输出：保留可安全解析的数据，同时标记查询不完整；
- 输出无法解析：保留上次成功快照并记录诊断信息；
- 数据量较大：解析、筛选和排序在后台完成，界面保持响应；
- 系统升级导致字段变化时，未知字段不得使整个查询崩溃。

## 10. UI/UX 设计规范

本节依据 UI UX Pro Max 的 minimal / Apple-style / accessibility 建议，并针对 macOS 原生工具进行取舍。

### 10.1 视觉方向

- 使用 SwiftUI/macOS 原生控件，避免模拟网页卡片；
- 字体使用系统 San Francisco 字体与 Dynamic Type，不引入网页字体；
- 颜色使用 macOS 语义色，主强调色建议使用系统蓝；
- 页面以系统背景、分隔线和层级排版建立结构，不依赖大面积阴影；
- 菜单栏面板和侧栏可适度使用系统 Material；正文表格保持清晰、稳定；
- 圆角克制：小型控件遵循系统默认，信息容器建议 8–12 pt；
- 间距以 4 pt 网格为基础，常用 8 / 12 / 16 / 24 pt；
- 使用 SF Symbols，图标风格、线宽和尺寸保持一致；不使用 Emoji 充当功能图标；
- 动效以状态过渡为主，建议 150–250 ms，不做悬浮放大、滚动劫持或装饰性长动画。

### 10.2 信息层级

1. 第一层：搜索框、当前是否被占用、进程名与端口；
2. 第二层：协议、状态、PID、用户、地址；
3. 第三层：文件描述符、路径、父进程及原始诊断信息；
4. 危险操作与浏览操作在位置和颜色上明显区分。

### 10.3 状态与颜色

- 正常监听：可使用系统绿色图标 + `监听` 文本；
- 活跃连接：系统蓝色图标 + 状态文本；
- 警告/权限不足：系统橙色图标 + 解释文本；
- 结束/强制结束：系统红色语义样式，仅用于危险动作；
- UDP、暂停、失败等状态必须同时提供图标或文字，不能仅依赖颜色；
- 浅色、深色、提高对比度与降低透明度模式下均需保持可读。

### 10.4 交互规范

- 可点击元素使用系统 hover、pressed 和 focus 反馈，不能因 hover 改变布局；
- 表格行单击选中，双击打开/展开详情，不将双击绑定危险操作；
- 破坏性操作使用系统确认对话框；默认焦点放在安全选项；
- 加载、空状态、无搜索结果、权限错误和查询错误使用不同文案与图标；
- 成功反馈简短且不遮挡工作流；失败反馈靠近触发动作并提供恢复路径；
- 尊重“减少动态效果”和“降低透明度”系统设置。

### 10.5 键盘与辅助功能

- `⌘F`：聚焦搜索；
- `⌘R`：立即刷新；
- `⌘,`：打开设置；
- `⌘W`：关闭窗口但不退出菜单栏应用；
- `⌘Q`：退出应用；
- `Space` 或 `Return`：打开选中项详情；
- 结束进程动作不设置容易误触的单键快捷键；
- 所有按钮、状态图标、表格列和菜单栏项提供 VoiceOver 标签与必要提示；
- Tab/方向键顺序与视觉顺序一致，无键盘陷阱；
- 支持系统文本缩放，关键文字不得截断到无法理解。

### 10.6 明确排除的视觉方案

UI UX Pro Max 搜索结果中以下方案不适合本产品，不采用：

- 金色品牌主色或餐饮风格衬线字体；
- 面向营销页的横向滚动叙事、巨型标题和 Bento 展示页；
- 大面积高透明玻璃、强渐变、视差或持续脉冲动画；
- 为追求“高级感”而牺牲表格密度、对比度或系统一致性。

## 11. 异常与边界场景

至少覆盖以下情况：

1. 指定端口当前无人占用；
2. 端口在两次刷新之间被占用或释放；
3. 查询完成后进程退出，用户随后点击结束；
4. PID 被系统快速复用；
5. 同一端口存在 IPv4/IPv6、TCP/UDP 或多进程记录；
6. 一个进程占用多个端口；
7. 结束进程后守护程序立即将其重启；
8. 用户无权结束目标进程；
9. 目标是系统关键进程；
10. `lsof` 执行超时、返回部分结果或格式异常；
11. 大量短连接导致列表频繁变化；
12. 设备休眠后恢复、用户快速切换、网络接口变化；
13. 菜单栏空间不足或系统隐藏状态项；
14. 用户开启减少动态效果、降低透明度或提高对比度；
15. 应用被强制退出后重新启动，不应误报上次操作成功。

## 12. 非功能需求

### 12.1 性能

- 常规机器、500 条以内网络记录时，首次可用结果目标为 1.5 秒内；
- 手动刷新后目标为 1 秒内更新结果，实际受 `lsof` 和系统负载影响；
- 查询和解析期间主窗口滚动、搜索输入与窗口操作保持流畅；
- 后台常驻时避免高频轮询，目标平均 CPU 占用需在原型阶段实测并设定上限；
- 内存不随刷新次数持续增长；
- 进程结束操作应有超时，不能无限等待。

### 12.2 稳定性

- 单次查询失败不导致应用崩溃；
- 数据解析错误应能定位到样本但避免记录不必要的敏感地址；
- 菜单栏组件与主窗口共享同一数据源，避免显示互相矛盾的快照；
- 应对快速重复刷新和重复点击结束操作进行去重或禁用。

### 12.3 安全与隐私

- 默认完全本地运行，不上传端口、IP、进程名或用户名；
- 不调用 Shell，不将用户搜索文本拼接为命令；
- 结束进程前按最新快照校验 PID 与端口关联；
- 管理员权限按次、按目标最小化使用，不保留用户密码；
- 日志默认不记录完整远端地址和命令行参数；
- 首次涉及管理员权限时解释用途；
- 发布包应进行代码签名与公证；分发渠道对权限模型的影响需验证。

### 12.4 兼容与本地化

- 最低支持 macOS 15 Sequoia，以使用现代 SwiftUI 菜单栏能力；
- 支持 Apple Silicon；不用支持 Intel Mac；
- 首版语言建议简体中文；代码与布局预留英文及长文本空间；
- 使用 Locale 友好的数字与时间格式；端口号、PID 不做千位分隔。

## 13. 埋点、日志与诊断

- P0 不接入第三方统计 SDK；
- 可记录仅存本地的诊断信息：查询耗时、退出码、解析失败计数、刷新时间、操作结果类型；
- 默认不记录完整连接列表、远端地址、用户名、可执行参数或用户搜索内容；
- 若未来加入崩溃上报或匿名统计，必须单独说明数据字段并提供明确开关。

## 14. P0 验收标准

1. 启动应用后，主窗口与菜单栏均能显示来自同一快照的端口信息。
2. 用户输入完整端口号后，可准确看到所有对应 TCP/UDP、IPv4/IPv6 占用记录。
3. 用户可按进程名、PID 搜索，并组合协议、状态和用户筛选。
4. 列表支持规定字段与数值端口排序，自动刷新后筛选和排序不丢失。
5. 点击记录可查看进程与端点详情，同一进程的其他端口可被识别。
6. 普通当前用户进程可在确认后收到 `SIGTERM`，应用随后重新查询并展示真实结果。
7. `SIGTERM` 未生效时，用户可在二次警告后选择 `SIGKILL`；取消不会执行任何信号。
8. 权限不足、进程已退出、端口被重新占用及查询失败时均有不同且可恢复的提示。
9. 点击菜单栏状态项可搜索、刷新、跳转主窗口和退出应用。
10. 关闭主窗口后菜单栏继续工作，选择退出后应用完全结束。
11. 支持浅色/深色、键盘导航、VoiceOver、减少动态效果和降低透明度。
12. 不通过 Shell 执行 `lsof`，用户输入不能改变实际命令参数结构。
13. 刷新与解析不阻塞主线程；连续运行与刷新不存在明显内存增长。
14. 在支持的 macOS 版本上完成签名构建与基本真机验证。

## 17. 技术方案阶段需验证的问题

1. 目标 macOS 版本中 `lsof -F` 的实际字段、分隔形式和 IPv6/UDP 输出差异；
2. 非管理员身份可见的数据范围，以及不同系统隐私设置下的变化；
3. 使用 App Sandbox、Hardened Runtime 和不同分发渠道时，查询及发送信号的可行性；
4. 管理员授权应使用何种受支持机制，以及是否需要特权辅助程序；
5. 如何可靠识别系统关键进程，避免基于名称的脆弱判断；
6. `lsof` 高频轮询的 CPU、能耗和大数据量表现；
7. 进程退出、PID 复用、守护进程重启时的校验策略；
8. SwiftUI `MenuBarExtra`、多窗口与 Dock 显隐在目标系统版本上的行为；
9. 应用图标、可执行路径和父进程信息应由哪些系统 API 补充；
10. 自动化测试如何构造 TCP/UDP、IPv4/IPv6、多进程共享及权限不足样本。
