# Port Viewer 易懂化与图形化改版需求文档

> 文档版本：v0.1（待确认初稿）  
> 文档状态：待产品确认  
> 更新日期：2026-07-20  
> 适用范围：macOS 主窗口为 P0，菜单栏与设置页为关联优化  
> 关联文档：`docs/requirements.md`  
> 说明：本文仅定义需求，不包含代码实现。所有存在产品歧义的内容统一放在第 14 节，并以 `【待确认】` 标识，便于手动修改。

## 1. 改版背景

当前版本已经能够正确查询并展示端口、进程和网络连接，也能完成搜索、筛选、刷新及结束进程等核心操作。但当前界面基本沿用了 `lsof` 的技术字段和表达方式，普通用户容易遇到以下问题：

1. 不理解 PID、TCP、UDP、IPv4、IPv6、本地端点、远端端点、LISTEN、ESTABLISHED、文件描述符和父进程等术语；
2. 能看到很多参数，却无法快速判断“哪个应用正在做什么”“这个端口能被谁访问”“是否需要处理”；
3. 主界面下方详情区以字段平铺为主，字段之间的关系不直观；
4. 技术状态使用英文缩写，用户需要额外搜索才能理解；
5. 同一信息同时出现地址、端口、协议、状态等多个维度，认知负担较高；
6. 颜色和图标能区分记录，但没有形成完整、可解释的视觉结论。

本次改版的重点不是删除专业信息，而是调整信息顺序：**先给出人能读懂的结论，再用图形解释关系，最后按需展开原始技术参数。**

## 2. 改版目标

### 2.1 核心目标

1. 非专业用户选中一条记录后，3 秒内能回答：
   - 是哪个应用或服务；
   - 它正在等待连接、存在连接活动，还是进行其他网络活动；
   - 只能由本机访问，还是可能被同一网络中的其他设备访问；
   - 结束该进程会产生什么影响。
2. 用户不需要理解英文状态码，也能看懂列表和详情区的主要结论。
3. 通过连接路径图直观展示“来源/目标—端口—应用”之间的关系。
4. 专业用户仍能查看完整原始字段，不降低现有排障能力。
5. 不根据有限数据做“安全/危险”“正常/恶意”等无法可靠证明的判断。

### 2.2 可衡量的验收目标

- 5 名首次使用且不了解网络端口的测试用户中，至少 4 名能在 10 秒内说出选中记录的用途概况和访问范围；
- 首屏主要区域不再强制用户阅读 PID、文件描述符或英文 TCP 状态码；
- 每个专业字段均可从界面内找到中文解释，无需跳转外部网页；
- VoiceOver 用户无需读取图形，也能获得与图形相同的文字结论；
- 完整技术信息相较当前版本不丢失。

## 3. 目标用户与使用层级

### 3.1 主要用户：非专业用户

- 只知道“某个应用占用了端口”，不了解网络协议；
- 关心应用在做什么、是否可能被其他设备访问、能否结束；
- 希望获得结论和操作建议，不希望先学习术语。

### 3.2 次要用户：初级开发者

- 知道端口号，但对 TCP 状态、监听地址和进程关系不熟悉；
- 需要快速找到 3000、5173、8080 等端口的占用者；
- 需要在操作前理解影响范围。

### 3.3 保留用户：专业开发者或运维人员

- 仍需要 PID、协议、IP 版本、原始地址、文件描述符、父进程和原始状态；
- 接受通过“技术详情”展开查看，不要求所有字段默认常驻。

## 4. 产品设计原则

1. **结论优先**：先显示“应用正在做什么”，再显示它对应的技术参数。
2. **逐层展开**：默认信息简洁；解释和专业字段按需展开。
3. **关系优于数值**：详情区优先用连接路径图展示实体关系，而非添加无意义的饼图或折线图。
4. **解释紧贴内容**：术语解释在字段附近通过问号按钮、说明文案或展开区提供。
5. **不夸大风险**：监听全部地址不等于已暴露到公网；必须使用“可能被访问”，并说明仍受防火墙和网络环境影响。
6. **颜色不是唯一信息**：图标、文字和形状必须与颜色共同表达状态。
7. **原生一致性**：继续使用 macOS 系统字体、SF Symbols、语义色、键盘操作和辅助功能。
8. **数据忠实**：界面可以合并展示重复技术记录，但原始记录必须可查看，不得因易懂化而篡改或丢失数据。

## 5. 改版范围与优先级

| 模块 | 需求 | 优先级 |
|---|---|---:|
| 主列表 | 用易懂字段替代默认技术列 | P0 |
| 主列表 | 技术术语即时解释 | P0 |
| 搜索与筛选 | 常用条件改为任务语言，技术条件放入“更多筛选” | P0 |
| 详情区 | 一句话结论与状态说明 | P0 |
| 详情区 | 连接关系图 | P0 |
| 详情区 | 访问范围解释 | P0 |
| 详情区 | “技术详情”折叠区，保留完整参数 | P0 |
| 详情区 | 操作影响与结束进程入口 | P0 |
| 状态文案 | 英文状态码转换为中文用户语言 | P0 |
| 无障碍 | 图形的文字等价描述、键盘与 VoiceOver 支持 | P0 |
| 菜单栏 | 将“监听”等文案同步为更易懂表达 | P1 |
| 首次引导 | 首次进入时介绍端口和三类活动 | P1 |
| 术语表 | 集中的“网络术语说明”页面 | P1 |
| 趋势图 | 历史端口或连接数量趋势 | P2，依赖历史数据能力 |

### 5.1 本次不做

- 不基于进程名判断应用是否恶意；
- 不承诺判断端口是否真正可从公网访问；
- 不进行公网 IP、地理位置、域名归属或威胁情报查询；
- 不增加网络扫描、抓包、防火墙修改或端口转发功能；
- 不为了图形化而展示缺乏决策价值的饼图、仪表盘或装饰性动画。

## 6. 新的信息层级

每条记录按三层信息展示：

### 第一层：一眼看懂

- 应用/服务名称与图标；
- 主要端口；
- 中文活动状态；
- 访问范围或连接对象；
- 是否属于其他用户或系统关键进程。

### 第二层：图形解释

- 一句话结论；
- 连接路径图；
- “这意味着什么”解释；
- 操作影响提示。

### 第三层：技术详情

- PID；
- TCP/UDP 与 IPv4/IPv6；
- 本地/远端原始地址和端口；
- 原始 TCP 状态码；
- 文件描述符；
- 可执行路径；
- 父进程；
- 更新时间；
- 同一进程的其他端口和连接。

## 7. 主窗口整体布局

### 7.1 建议结构

```text
┌──────────────────────────────────────────────────────────────────────────┐
│ Port Viewer      [搜索应用或端口…]                     [暂停] [刷新] [设置] │
├──────────────┬───────────────────────────────────────────────────────────┤
│ 全部活动     │  等待连接 24     连接活动 61     其他网络活动 8             │
│ 等待连接     │  数据刚刚更新                                  [什么是端口?] │
│ 连接活动     ├───────────────────────────────────────────────────────────┤
│ 其他网络活动 │  应用/服务     端口     正在做什么          访问范围/连接到   │
│              │  node          3000     等待连接            仅这台 Mac       │
│              │  Safari        53124    连接已建立          142.250…:443     │
│              │  ...                                                       │
│              ├───────────────────────────────────────────────────────────┤
│              │  node 正在通过端口 3000 等待这台 Mac 上的应用连接           │
│              │                                                           │
│              │  [这台 Mac] ─────→ [端口 3000] ─────→ [node]               │
│              │                                                           │
│              │  仅本机可访问  ·  当前没有正在传输数据的含义                │
│              │  [技术详情 ▸]                         [结束这个进程…]        │
└──────────────┴───────────────────────────────────────────────────────────┘
```

### 7.2 区域职责

- **侧栏**：使用用户任务语言区分活动，不直接要求用户理解 TCP/UDP；
- **概况区**：展示活动数量与数据新鲜度，支持查看解释；
- **主列表**：用于扫描和选择，默认只放可以支持判断的字段；
- **详情区**：用于解释单条记录和执行操作；
- **技术详情**：默认折叠，承载当前已有的完整参数。

## 8. 主列表需求

### UX-001 默认列

默认列表列调整为：

| 默认列 | 内容 | 示例 |
|---|---|---|
| 应用/服务 | 图标、进程名；必要时显示“其他用户”或“系统进程”标记 | node |
| 端口 | 以数字为主；无明确端口时显示可理解文案 | 3000 |
| 正在做什么 | 中文状态，不显示英文代码作为主文案 | 等待连接 |
| 访问范围/连接到 | 监听记录显示访问范围；活动连接显示对端；UDP 按数据情况显示 | 仅这台 Mac |

以下字段不再默认常驻主列表，但必须保留在技术详情中：PID、协议、IP 版本、用户、文件描述符、完整本地端点、原始 TCP 状态。

### UX-002 列表中文状态

状态转换遵循第 11 节的映射。中文文案为主，原始英文状态只能作为辅助文案或技术详情出现，例如：

- 主文案：`连接已建立`
- 辅助文案或提示：`TCP 状态：ESTABLISHED`

### UX-003 访问范围标签

监听记录至少区分：

- `仅这台 Mac`；
- `可能被其他设备访问`；
- `访问范围暂不确定`。

活动连接显示：

- `连接到 <远端地址>:<端口>`；
- 无法获得对端时显示 `连接对象未知`。

不得将“可能被其他设备访问”简写成“公网开放”或“危险”。

### UX-004 参数解释入口

- 列名中存在可能陌生的概念时，提供可点击且可键盘聚焦的问号按钮；
- 指针悬停可显示简短提示，点击后显示完整解释；
- 不允许只依赖悬停，因为键盘和 VoiceOver 用户也需要访问；
- 解释内容必须包含“它是什么”和“为什么用户可能需要关心”；
- 不在每一行重复放置问号，避免列表噪声。

### UX-005 筛选器易懂化

- 搜索框提示改为 `搜索应用名称或端口，例如 3000`；
- 搜索仍支持 PID，但把 `按 PID 搜索` 作为搜索帮助中的专业能力，不要求普通用户理解；
- 默认显示的常用筛选为：
  - `活动类型`：全部、等待连接、连接已建立、正在建立/关闭、其他网络活动；
  - `访问范围`：全部、仅这台 Mac、可能被其他设备访问、范围不确定；
  - `归属`：全部、我的应用/服务、其他用户；
- TCP/UDP、IPv4/IPv6、原始 TCP 状态放入 `更多筛选`，每项附带第 11 节中的短解释；
- 原始状态的选项采用 `连接已建立（ESTABLISHED）` 格式，不只显示英文代码；
- 已启用条件以可关闭标签展示，例如 `仅这台 Mac ×`、`TCP ×`，并提供 `清除全部`；
- 筛选能力继续支持组合，结果为空时用用户当前选择生成提示，例如：`没有找到“仅这台 Mac”的 UDP 活动`；
- 常用筛选与“更多筛选”中的技术条件必须作用于同一份数据，不能出现含义不一致的重复控件。

### UX-006 记录合并与重复

- 主列表允许将“同一进程、同一协议、同一活动含义”的底层记录归为一项易读活动；监听活动仅在访问范围含义一致时归组；
- 多端口监听需要显示 `N 个服务端口`；多条活动连接需要分别显示连接数、连接目标数和本机连接端口数；
- 主列表使用端口芯片、数量徽标和迷你拓扑表达这些数量关系，不以摘要句作为唯一表达；
- 同一应用的多个进程使用相同监听端口时，默认收敛为一个服务组；应用列可展开查看 PID，结束进程前必须选择具体进程；
- 远端地址不同的活动连接不能表述成一条连接，但可以归入同一个进程活动摘要，并在技术详情中保留每条独立连接；
- 相同本机端口连接多个目标时，需要明确说明“相同端口不代表同一连接”；多个本机连接端口连接同一目标时，需要明确说明“这些端口不表示对外开放了多个服务”；
- 只有一项包含多个等价底层记录且没有更有意义的端口/连接摘要时，才显示 `包含 N 条技术记录`；
- 访问范围不同的监听地址不能错误合并为同一访问范围；
- 合并规则不得影响搜索、筛选和结束进程前的实时校验。

## 9. 下方详情区需求

### UX-007 未选择记录时

当前“选择一条记录查看详情”的空状态改为教学型空状态：

- 标题：`选择一项，我会解释它正在做什么`；
- 说明：`你会看到应用、端口、连接关系和访问范围。专业参数仍可在“技术详情”中查看。`；
- 可选辅助入口：`先了解什么是端口`。

### UX-008 一句话结论

选中记录后，详情区顶部必须先展示一条由当前数据生成的自然语言结论，不得先显示参数表。

示例：

- 监听且仅本机：`node 正在通过端口 3000 等待这台 Mac 上的应用连接。`
- 监听且绑定全部地址：`Python 正在通过端口 8000 等待连接，同一网络中的其他设备可能也能访问它。`
- 已建立连接：`Safari 与 142.250.72.14 的 443 端口之间已建立连接。`
- UDP 且无远端：`mDNSResponder 正在使用 UDP 端口 5353 发送或接收无固定连接的数据。`
- 无法完整判断：`该进程正在进行网络活动，但目前的信息不足以确定连接对象。`

结论文案必须使用“正在”“可能”“暂不确定”等与数据确定性匹配的措辞。

### UX-009 连接关系图

详情区中间展示小型连接路径图。它是主信息，而非装饰图。

#### A. 等待连接（TCP LISTEN）

```text
[访问来源] ─────→ [这台 Mac：端口 3000] ─────→ [node]
```

- 仅回环地址时，“访问来源”显示 `这台 Mac`；
- 通配或非回环地址时，显示 `这台 Mac / 同一网络设备`；
- 对“同一网络设备”的连线使用虚线，并标注 `可能可访问`；
- 图下说明：`实际能否访问还会受到 macOS 防火墙、路由器和应用设置影响。`

#### B. 连接活动（存在远端端点）

```text
[Safari] ←────→ [本机临时端口 53124] ←────→ [142.250.72.14：443]
```

- 双向连线只表示两端存在连接关系，不宣称当前正在传输数据，也不推断连接最初由哪一端发起；
- 本机临时端口弱化显示，应用和远端连接对象为视觉重点；
- 若无法识别域名，只显示原始地址，不擅自补充网站名称或地理位置。

#### C. UDP 或无固定连接

```text
[mDNSResponder] ←──→ [UDP 端口 5353] ←──→ [可能的通信对象]
```

- 使用双向或无固定方向的视觉表达；
- 明确说明 `UDP 不保持“已连接/未连接”状态，因此这里没有 TCP 那样的连接状态。`；
- 若数据中存在明确远端地址，则用实际地址替代“可能的通信对象”。

#### D. 图形交互

- 节点可聚焦；聚焦或点击后显示对应解释；
- 不要求拖拽，不使用力导向网络图，避免布局不稳定；
- 不自动播放动画；刷新时只做 150–250ms 的淡入淡出；
- 开启“减少动态效果”时取消非必要动画；
- 图形必须同时提供一段可复制的文字等价描述。
- 多端口、多目标或多进程共享端口时，节点内部使用分支条目展示最多 3 项，其余项目显示数量并保留在技术详情中；不得把聚合节点画成一条单独连接。

### UX-010 “这意味着什么”说明

关系图下方展示 1–3 条解释卡片或短句，内容根据记录类型变化：

- `仅本机可访问`：其他设备通常不能直接连接这个地址；
- `可能被其他设备访问`：同一网络中的设备可能具备访问条件；是否真正可访问仍取决于防火墙和网络配置；
- `连接已建立`：说明两端已建立连接并具备交换数据的条件，但不代表此刻一定在传输数据；
- `刚刚结束`：连接已经结束，系统正在短暂保留记录；
- `其他用户进程`：当前版本不能直接结束它；
- `关键系统进程`：结束可能影响系统功能，强制结束已禁用。

说明中禁止使用无法从当前数据证明的文案，例如：

- `该连接是安全的`；
- `该应用正在上传文件`；
- `这个端口已暴露到互联网`；
- `发现病毒或异常行为`。

### UX-011 技术详情折叠区

- 标题统一为 `技术详情`；
- 默认折叠；
- 展开状态在切换选中项时的行为见 `【待确认-04】`；
- 展开后以分组方式展示，不再使用单个密集网格平铺所有字段。

建议分组：

1. **进程信息**：进程名、PID、用户、父进程、可执行路径；
2. **连接信息**：TCP/UDP、IPv4/IPv6、本地地址与端口、远端地址与端口、原始状态；
3. **系统信息**：文件描述符、记录更新时间、底层记录数量；
4. **同一进程的其他活动**：其他端口和连接，可点击切换到对应记录。

每个技术字段旁提供一句短解释。长路径和地址支持复制，默认用中间截断而不是隐藏完整内容。

### UX-012 操作区

- 主按钮继续使用 `结束进程…`，不使用 `关闭端口` 或 `Kill`；
- 按钮附近显示影响摘要：`结束后，这个应用使用的其他 N 个端口或连接也会关闭。`；
- 非当前用户进程的按钮禁用，并解释权限原因；
- 关键系统进程显示醒目但克制的警告；
- 确认框和操作后重新校验流程沿用原需求，不因界面简化而降低安全性；
- 视觉上，红色只用于破坏性操作和错误，不用于普通监听状态。

### UX-013 数据刷新时的详情稳定性

- 自动刷新不得让关系图频繁重排；
- 记录仍存在时保持选中和详情展开状态；
- 记录消失时显示：`这项网络活动已结束`，并保留最后一次信息 3–5 秒供用户理解；
- 若同一端口立即被新进程占用，明确显示：`原活动已结束，但端口现在由另一个进程使用。`；
- 显示相对时间 `刚刚更新`，精确时间放入技术详情。

## 10. 概况区与侧栏需求

### UX-014 概况指标

概况区建议展示：

- `等待连接`：当前 TCP 监听记录数；
- `连接活动`：当前存在远端地址的非监听 TCP 记录数，包含建立中、已建立和关闭中的连接；
- `其他网络活动`：UDP 及无法归入前两类的记录数。

每个指标均提供说明。数量只用于概览，不将数量大小解释为风险高低。

查询耗时如 `23 ms` 默认移入技术状态提示或技术详情，不在普通用户首屏常驻。

### UX-015 侧栏命名

建议将当前侧栏命名调整为：

| 当前名称 | 建议主名称 | 辅助解释 |
|---|---|---|
| 全部 | 全部活动 | 查看所有端口和网络连接 |
| 监听端口 | 等待连接 | 应用正在等待其他程序连接 |
| 活跃连接 | 连接活动 | 应用与另一个地址之间存在建立中、已建立或关闭中的连接 |
| UDP | 其他网络活动 | 包含不保持连接状态的 UDP 活动 |

“UDP”仍可作为辅助标签显示，避免专业用户无法定位。

## 11. 术语与状态文案规范

### 11.1 字段名称与解释

| 技术字段 | 用户可见名称 | 界面内解释 |
|---|---|---|
| Process / Command | 应用/服务 | 正在使用网络的应用、后台服务或系统进程 |
| PID | 进程编号 | 系统为每个正在运行的进程分配的临时编号，主要用于精确识别和排障 |
| Port | 端口 | 应用在这台 Mac 上用于接收或发送网络数据的“编号入口” |
| TCP | 可靠连接方式 | 会建立连接并确认数据送达，网页和本地开发服务经常使用 |
| UDP | 快速传输方式 | 不保持固定连接状态，常用于发现设备、音视频或实时通信 |
| IPv4 / IPv6 | 地址格式 | 两种网络地址格式，通常不需要手动处理 |
| Local endpoint | 这台 Mac 的地址和端口 | 当前进程在本机使用的网络地址与端口 |
| Remote endpoint | 连接对象 | 这条连接另一端的地址与端口；仅凭该字段不能判断此刻是否有数据传输 |
| User | 归属用户 | 启动该进程的 macOS 用户 |
| File descriptor | 系统连接编号 | 进程内部用于标识这条网络资源的编号，主要用于高级排障 |
| Parent process | 启动来源 | 启动当前进程的上一级进程 |
| Executable path | 程序位置 | 当前进程对应程序文件在磁盘上的位置 |
| Updated at | 数据更新时间 | 应用上次确认这条记录存在的时间 |

### 11.2 TCP 状态映射

| 原始状态 | 主文案 | 用户解释 |
|---|---|---|
| LISTEN | 等待连接 | 应用开放了一个端口，正在等待其他程序连接 |
| ESTABLISHED | 连接已建立 | 双方具备交换数据的条件，但不代表此刻一定有数据传输 |
| SYN_SENT | 正在发起连接 | 这台 Mac 正在尝试连接另一端 |
| SYN_RECEIVED | 正在确认连接 | 已收到连接请求，正在完成建立连接 |
| TIME_WAIT | 刚刚结束 | 连接已经关闭，系统暂时保留记录以处理延迟数据 |
| CLOSE_WAIT | 等待应用关闭 | 另一端已结束连接，本机应用还在完成关闭流程 |
| FIN_WAIT_1 / FIN_WAIT_2 | 正在关闭 | 本机已开始关闭连接，正在等待另一端确认 |
| LAST_ACK | 正在完成关闭 | 连接即将结束，正在等待最后确认 |
| CLOSING | 正在关闭 | 双方都在关闭连接 |
| CLOSED | 已结束 | 连接已经关闭 |
| 无状态且为 UDP | 正在使用 | UDP 不保持 TCP 式连接状态 |
| 未识别状态 | 其他状态 | 显示原始状态码，并说明“这是系统返回的技术状态” |

### 11.3 地址与访问范围映射

| 底层地址特征 | 用户标签 | 必须附带的限定 |
|---|---|---|
| `127.0.0.1`、`::1`、`localhost` | 仅这台 Mac | 通常只能由本机应用访问 |
| `*`、`0.0.0.0`、`::` | 可能被其他设备访问 | 仍受系统防火墙、路由器和应用设置影响 |
| 明确的非回环本地地址 | 可能被其他设备访问 | 只说明绑定了该网络接口，不代表公网一定可达 |
| 地址缺失或无法解析 | 访问范围暂不确定 | 可在技术详情查看原始值 |

## 12. 视觉与交互规范

### 12.1 状态视觉

- `等待连接`：青色或绿色圆点 + 文字；
- `连接活动`：蓝色双向箭头 + 具体中文状态；
- `正在建立/关闭`：灰蓝色进度图标 + 文字；
- `可能被其他设备访问`：橙色轮廓网络图标 + 限定文案，不使用红色；
- `其他用户/关键系统进程`：橙色锁或警告图标 + 文字；
- `错误/结束进程`：红色，仅用于真实错误与破坏性操作。

最终颜色使用 macOS 语义色并同时验证浅色、深色及“增强对比度”模式。

### 12.2 排版

- 一句话结论使用 headline 或 title3，确保是详情区第一视觉焦点；
- 正文使用系统 body/callout，不使用大段等宽字体；
- 等宽字体只用于端口、PID、IP 地址、文件描述符等技术值；
- 单行解释尽量控制在 30 个中文字符以内，详细说明允许换行；
- 技术详情字段按组分隔，避免 4 列以上的密集网格。

### 12.3 关系图

- 使用固定的 3 段水平路径，不使用随机布局；
- 节点使用圆角矩形，至少包含图标、主名称和必要的辅助值；
- 实线表示已知关系，虚线表示“可能”或“无法完全确认”；
- 箭头旁用短文案标注 `等待连接`、`连接已建立` 或 `可能可访问`；
- 窗口较窄时切换为纵向路径，不产生横向滚动；
- 图形高度建议 96–140 pt，不挤占主列表主要空间。

### 12.4 帮助浮层

- 简短解释不超过 2 句；
- 浮层不遮挡当前选中记录和关键操作；
- 可通过 `Esc` 关闭；
- 点击外部可关闭；
- 支持文字选择时，不因指针移出立即消失。

## 13. 无障碍与验收标准

### 13.1 无障碍要求

- 每个状态图标都有包含状态和数量的辅助功能标签；
- 关系图作为一个组合元素时，VoiceOver 首先朗读完整结论，再允许进入节点；
- 虚线、实线和颜色差异必须有文字说明；
- 问号按钮、技术详情和结束进程按钮均可使用键盘访问；
- 焦点顺序：结论 → 关系图/文字描述 → 解释 → 技术详情 → 操作；
- 支持“减少动态效果”“增强对比度”和系统字体缩放；
- 不要求用户通过悬停才能获得必要信息。

### 13.2 功能验收用例

1. **本机监听**：当地址为 `127.0.0.1:3000` 且状态为 LISTEN 时，显示“等待连接”“仅这台 Mac”和本机来源路径图。
2. **全部地址监听**：当地址为 `*:8000` 时，显示“可能被其他设备访问”，并同时显示防火墙限定说明。
3. **已建立连接**：当状态为 ESTABLISHED 且存在远端端点时，显示应用与远端之间的连接关系图，且不暗示此刻一定在传输数据。
4. **UDP**：无 TCP 状态时不显示错误或空白状态，显示“正在使用”及 UDP 解释。
5. **未知状态**：系统返回未映射状态时，显示“其他状态”和原始状态码，不崩溃、不错误翻译。
6. **其他用户进程**：显示归属提示，结束按钮不可直接执行，并能解释原因。
7. **技术详情**：当前版本已有的所有字段均可找到、选择和复制。
8. **自动刷新**：记录持续存在时图形不重排，选中状态不丢失。
9. **记录结束**：选中记录消失时显示易懂的结束提示，不突然切回完全空白详情。
10. **VoiceOver**：关闭视觉后，用户仍能从文字朗读获得应用、端口、状态、连接对象和访问范围。

### 13.3 文案验收

- 主界面默认不以 LISTEN、ESTABLISHED、PID、FD、endpoint 等词作为主要信息；
- 出现技术词时，同一上下文可访问中文解释；
- 不出现“关闭端口”这种会误导实际操作对象的文案；
- 不出现没有证据支持的安全结论；
- 所有“不确定”情况均明确表达信息边界。
